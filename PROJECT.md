# 돈길 — 개발 로드맵 (Claude Code 작업용)

> 재미있고 명랑하게, 그러나 유익하게. **경제 문맹 퇴치.**
> 이 문서는 클로드 코드와 바로 작업을 시작하기 위한 실행 계획서입니다.
> 각 Task는 그대로 복사해 클로드 코드에 시키면 되는 크기로 쪼개져 있습니다.

---

## 0. 이 문서 쓰는 법

1. 먼저 **Phase 0**으로 프로젝트를 세팅한다.
2. 이후 Task를 위에서부터 하나씩 클로드 코드에 던진다. (한 번에 하나씩!)
3. 각 Task에는 "완료 기준"이 있다. 그게 충족되면 다음 Task로 넘어간다.
4. 막히면 해당 Task만 다시 시키거나 쪼갠다.
5. **MVP는 Phase 1까지다.** Phase 2~3은 출시 후 붙인다.

---

## 1. 제품 원칙 ★ 항상 기억할 것

이 셋이 모든 결정의 기준이다. 기능을 추가할지 말지 헷갈리면 여기로 돌아온다.

### 진입장벽 제로
- **회원가입 없이 바로 첫 레슨.** 가입은 "진도를 저장하고 싶을 때" 선택사항으로만.
- **30초 안에 첫 성공("맞혔다!").** 첫 문제는 누구나 맞히는 쉬운 문제로.
- 한 화면에 한 가지만. 한 레슨 3분, 한 문제 한 화면.
- 전문용어가 나오면 즉시 풀어준다 (밑줄 → 탭하면 말풍선 설명).

### 재미 (명랑함)
- 마스코트 캐릭터가 안내하고 칭찬한다. (예: 엽전/돼지저금통 캐릭터)
- 친근한 말투. 격려는 듬뿍 ("오, 좋은데요?", "거의 다 왔어요!").
- 틀려도 혼나는 느낌 0. ("괜찮아요, 같이 다시 봐요") + 바로 쉬운 풀이.
- 정답 시 작은 축하(컨페티), 연속 출석 불꽃, 레벨업 팡파레.
- 명랑한 색감, 둥근 모서리, 일러스트.

### 유익함
- 모든 개념을 "내 얘기"로. ("월급 250만 원이면 4대 보험으로 약 OO원이…")
- 배운 걸 바로 써먹는 계산기/도구를 붙인다.
- 용어 사전을 던지지 말고, 맥락 속에서 한 입씩.

## 2. 기술 스택 (MVP 기준 — 백엔드 없이 빠르게)

- **프레임워크:** Next.js (App Router) + TypeScript
- **스타일:** Tailwind CSS
- **애니메이션:** Framer Motion (축하/전환 효과)
- **아이콘:** lucide-react
- **상태/저장:** Zustand + `localStorage` (MVP는 계정 없이 브라우저에 진도 저장)
- **콘텐츠:** `/content` 폴더에 TypeScript 객체로. 스키마 검증은 Zod.
- **나중에(Phase 3):** Supabase(계정·동기화), Leaflet+OpenStreetMap(지도), 공공 API

> MVP는 서버 없이 전부 클라이언트에서 돈다. 계정/동기화/지도는 Phase 3에서 붙인다.

## 4. 데이터 모델 (lib/schema.ts에 Zod로)

```ts
// 콘텐츠
type Level   = { id: string; order: number; title: string; description: string; unitIds: string[] }
type Unit    = { id: string; levelId: string; order: number; title: string; lessonIds: string[] }
type Lesson  = {
  id: string; unitId: string; order: number;
  title: string; durationMin: number;
  intro: string;            // 마크다운, 한 입 설명
  questions: Question[];
  glossary: string[];       // 이 레슨에서 등장하는 용어 id
}

// 문제 (type으로 구분되는 유니온)
type Question =
  | { type: 'ox';     prompt: string; answer: boolean;        explanation: string }
  | { type: 'choice'; prompt: string; options: string[]; answerIndex: number; explanation: string }
  | { type: 'fill';   prompt: string; answer: string[];       explanation: string }  // 정답 후보 배열

// 용어
type GlossaryTerm = { id: string; term: string; short: string; full?: string }

// 사용자 진도 (localStorage)
type Progress = {
  completedLessonIds: string[];
  currentLessonId: string | null;
  xp: number;
  streak: { count: number; lastDate: string };   // YYYY-MM-DD
  hearts: number;
  wrongQuestionIds: string[];                     // 복습용(Phase 2)
}
```

> 세금·연금 수치(공제 한도·세율 등)는 매년 바뀐다. **콘텐츠에 하드코딩하지 말고** `content/rates.ts` 같은 파일로 빼서 "올해 기준" 변수로 관리한다.

## 5. 단계별 로드맵

### Phase 0 — 프로젝트 셋업

**Task 0.1 — 스캐폴딩**
- 내용: 위 스택으로 Next.js(App Router)+TS+Tailwind 프로젝트를 만들고, 위 폴더 구조의 빈 골격을 잡아라. Framer Motion, lucide-react, zustand, zod 설치.
- 완료 기준: `npm run dev`로 빈 홈 화면이 뜬다.

**Task 0.2 — 디자인 토큰 & 톤 설정**
- 내용: 명랑하고 따뜻한 팔레트(밝은 단일 액센트 + 부드러운 배경), 둥근 모서리, 친근한 본문 폰트를 Tailwind 설정에 잡아라. 공용 Button/Card 컴포넌트와 친근한 마이크로카피 톤 가이드(짧고 격려하는 말투)를 정해라.
- 완료 기준: Button/Card 컴포넌트가 일관된 명랑한 스타일로 렌더된다.

---

### Phase 1 — MVP (홈 + 학습 레벨1~2 + 계산기 1 + 스트릭)

목표: **가입 없이 들어와서, 3분 만에 한 레슨을 끝내고, 내일 또 오고 싶게.**

**Task 1.1 — 스키마 + 진도 로직**
- 내용: `lib/schema.ts`에 위 데이터 모델을 Zod로 정의. `lib/progress.ts`와 `store/useProgress.ts`에 localStorage 기반 진도/스트릭/하트 로직 구현(완료 처리, XP 적립, 날짜 비교로 스트릭 증가/끊김).
- 완료 기준: 함수 단위로 진도 저장·로드·스트릭 계산이 동작한다(간단 테스트 포함).

**Task 1.2 — 레슨 플레이어 (핵심!)**
- 내용: `app/learn/[lessonId]`에 레슨 플레이어. intro(한 입 설명) → 문제들을 한 문제 한 화면으로. OX·객관식·빈칸 3타입 지원, 상단 진행바. **정답이면 컨페티+격려, 오답이면 부드러운 멘트+즉시 풀이.** 마지막에 "잘했어요!" 요약 + XP 획득.
- 완료 기준: 더미 레슨 1개로 처음부터 끝까지 풀 수 있고, 완료 시 진도가 저장된다.

**Task 1.3 — 용어 풀이 말풍선**
- 내용: 본문 텍스트에서 용어를 밑줄로 표시하고, 탭하면 `glossary.ts`의 short 설명이 말풍선으로 뜨게. (전문용어 즉시 해소 = 진입장벽 제로의 핵심)
- 완료 기준: 레슨 안 용어를 탭하면 설명이 뜬다.

**Task 1.4 — 학습 경로(트리) 화면**
- 내용: `app/learn`에 듀오링고식 경로. 노드 상태 3종(완료=체크, 현재=강조, 잠김=자물쇠), 잠금 해제 로직(앞 레슨 완료 시 다음 열림), 유닛 헤더.
- 완료 기준: 레슨을 깨면 다음 노드가 열리고, 노드를 누르면 해당 레슨으로 간다.

**Task 1.5 — 홈 대시보드**
- 내용: `app/page.tsx`에 홈. **"오늘의 학습"(이어서 풀기) 카드**를 가장 크게, 그 위에 스트릭(불꽃)+XP, 아래에 "오늘의 한 입"(경제 용어 1개). 마스코트가 인사.
- 완료 기준: 들어오면 0초 만에 "오늘 뭐 할지"가 보이고, 카드 한 번으로 학습 시작.

**Task 1.6 — 마스코트 컴포넌트**
- 내용: `components/mascot`에 마스코트 캐릭터(SVG/일러스트)와 말풍선. 정답·오답·완료·빈 화면 상황별 대사 세트. 과하지 않게, 사랑스럽게.
- 완료 기준: 홈·레슨에서 상황에 맞는 마스코트 멘트가 나온다.

**Task 1.7 — 레벨 1~2 콘텐츠 입력**
- 내용: `content/`에 레벨 1(돈의 기초)·레벨 2(세금과 공제)의 유닛·레슨·문제를 채워라. **말투는 친근하게, 예시는 "내 월급" 같은 실생활로, 첫 레슨 첫 문제는 누구나 맞힐 만큼 쉽게.**
- 완료 기준: 레벨 1~2를 실제로 끝까지 풀 수 있다.

**Task 1.8 — 계산기 1: 실수령액**
- 내용: `app/tools/take-home`에 월급/연봉 입력 → 4대 보험·소득세 추정 차감 → 실수령액 출력. 결과 옆에 "이 숫자가 왜?" → 관련 레슨 링크. **"추정치예요, 정확한 건 회사/국세청 확인" 안내 문구 필수.**
- 완료 기준: 금액을 넣으면 실수령액과 항목별 차감이 보기 좋게 나온다.

**Task 1.9 — 게스트 모드 마감 점검**
- 내용: 전체 흐름을 가입·로그인 없이 끝까지 되게 확인. 진도는 localStorage에. 첫 방문자 온보딩은 "바로 첫 레슨"으로 최소화.
- 완료 기준: 처음 온 사람이 클릭 몇 번으로 첫 레슨을 끝낸다. **여기까지가 출시 가능 MVP.**

### Phase 2 — 습관 강화 (복습·도구·꾸미기)

- **Task 2.1 복습 시스템:** 틀린 문제(`wrongQuestionIds`)를 며칠 뒤 다시 내는 간격 반복 + 데일리 랜덤 퀴즈 5문제.
- **Task 2.2 계산기 확장:** 연말정산 환급 예측, 연금저축 세액공제, 복리 계산기. 각 결과를 관련 레슨에 연결.
- **Task 2.3 용어 사전 페이지:** "내가 배운 용어" 모음 + 검색.
- **Task 2.4 성취/축하 강화:** 배지, 레벨업 팡파레, 학습 캘린더(잔디), 스트릭 보호 알림 안내.
- **Task 2.5 실력 진단 테스트:** 처음 온 사람이 레벨을 건너뛸 수 있게.

### Phase 3 — 확장 (계정·지도·심화)

- **Task 3.1 Supabase 계정:** 가입/로그인 + 진도 클라우드 동기화(게스트→계정 이전 포함).
- **Task 3.2 지도 섹션:** Leaflet+OpenStreetMap. 우리 동네 부동산 실거래가(국토부 API) 또는 글로벌 경제 '날씨'. 데이터엔 캐싱.
- **Task 3.3 심화 콘텐츠:** 레벨 3(연금)·4(굴리기 입문)·5(신용/빚)·6(보험). **레벨 4는 원리 이해까지만, 종목 추천·투자 조언 금지.**
- **Task 3.4 알림/리텐션:** 매일 학습 리마인더(웹 푸시), 스트릭 위험 알림.

## 7. 주의사항 (계속 지킬 것)

- **종목 추천·투자 조언 금지.** 레벨 4는 "원리 이해"까지만. 법적 리스크 + 미션과 충돌.
- **세금·연금 수치는 매년 변한다.** `content/rates.ts`로 분리해 "올해 기준"으로 관리.
- **계산기는 추정치.** 항상 "정확한 건 국세청/회사/전문가 확인" 안내.
- 콘텐츠가 곧 제품의 질이다. 기능보다 **첫 몇 레슨의 말투와 쉬움**에 공을 들일 것.
